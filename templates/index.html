Variables
It's time to get into the meat and potatoes of Python by learning about variables. Imagine that you have a container that can hold different objects like books, a laptop, or even your homework. In Python, a variable is that backapck which can hold different kinds of data such as numbers, words, or even lists of data. Let's start with a really simple example. age = 15 print(age) #returns 15 in console The number 15 on the right of the = symbol is stored in age on the left. Then, whenever age is placed within the parentheses in print() and run our code, it's printed below in the console. Cool, right? Okay, not the most exciting result, but understanding this now will make your understanding clearer in the future. We can also change the value of age at any point like this: age = 15 print(age) age = 16 # Change the value of age print(age) Now when we run our code, 15 is printed in the console like before. Then, age is reassigned a value of 16 and is printed in the console. However, it's important to understand that age no longer contains the value 15. We changed it! I should point out that modifying variables this way can create problems for us later on. I'll explain this in the next section.

Variable Scope

In the previous section, you learned how to declare a simple variable and then reassign that variable a different value in a breif example. Declaring variables this way is fine for small applications, but in larger applications this approach can create problems. This leads us to the concept of scope. Python scopes are pretty confusing to approach all at once, so instead lets just focus on two concepts: global and local scopes. Think of scope like being at school, and the variables you want to assign are like the supplies you take with you in your backpack. The supplies you have in your backpack are global, because you can take them with you to class, to the cafeteria, outside to the commons area (you get the idea). They're accessible anywhere. Then you have certain materials that are available to you but only in certain settings. For example, in a PE class you have soccer balls provided to you, but you can't take them home with you. The soccer ball is a local variable. It's only accessible only in PE class, and nowhere else. Let's translate this into python code. backapck = "All materials in my backpack" def classroom(): phys_ed_classroom = "soccer balls" print("Getting stuff from backpack:", backapck) print("Getting stuff from PE Class: ", phys_ed_classroom) classroom() # call the function print("Getting stuff from backpack:", backpack) Feel free to copy and paste this and run it yourself. You will notice three print statements inside the console. Also, notice that the response from print("Getting...:", backpack) is printed twice. This is because backpack is declared outside of the function classroom. You can access your backpack and the contents inside it anywhere, where as you can't access things from your PE class except when you're in PE.In fact, try replacing backpack in the final print statement (the one outside of the function) with phys_ed_classroom and run it. You should see an error message that states that phys_ed_classroom is not defined. Whenever you declare a variable globally, it "comes into existence immediately." That's pythonic for what happens when you declare anything outside a function. Think about your backpack again. When you put it over your shoulders and head to class, everything inside of it is available to you right away. You can take out your pencils, pens, laptop, etc from any location at any time. It's okay if this stuff is abstract right now. Learning to code isn't a race, it's a marathon. Pace yourself!

Integers

You've seen an integer already. age = 13 # This is an integer An integer is a whole number. Integers have no decimal and consist of all positive numbers, zero, and negative numbers.

What's the big deal?

Being able to distinguish between data types will help us determine the appropriate type of variable we want to create whenever desinging our application. Say you're designing a simple app that categorizes what generation you fall into based on your age. The program prompts you to type in how old you are, and returns which generation you fall into (Baby boomer, Gen X, Millenial, Gen Z, Gen Apla). age = int(input("Enter your age: ")) # Get age from the user if age >=60: print(f"You are a baby boomer") elif age < 60 and age > 41: print("You're Gen X") elif age >= 30 and age <= 41: print("You're a millenial") elif age >= 20 and age <= 29: print("It's giving Gen Z") else: print("Gen Aplha") It makes the most sense to type in an integer, because we don't think of people's age as fractional values. Well, maybe your younger sibling or cousin who insists 5.5 is a real age and you're totally wrong, but in more normal circumstances, age is just an integer.

Floats

Floats, on the other hand, are numbers with fractional values. Floats are best utilized when we need a more precise numerical representation, like measuring length, height or performing calculations that might have fractional values. kilometers = 3.1 In this case, if you ran 3 kilometers and your friend ran 3.1 kilometers, your friend ran further by .1 kilometers. It's important to pause and consider which type of data you're going to work with when developing.

Strings

Remember when we learned about scope? You saw an example of a string in that snippet of code. In Python, strings are variables that contain a word, or words enclosed in single quotes '' or double quotes "". There's no rule on whether you use single or double quotes, just be consistent. first_name = "Luigi" last_name = "Mario" The above variables are strings. Strings are commonly used for things like people's names or storing passwords with special combinations of alphanumerical values and special characters. first_name = "Luigi" last_name = "Mario" The variable first_name contains the name "Luigi", and similarly with last_name. Like integers and floats, we can print the value of these variables with the print() function, but with strings, we can apply a cool trick called concatenation, which just means we can join strings together. first_name = "Luigi" last_name = "Mario" print(first_name + " " + last_name) # concatenates first_name and last_name with an empty space between them. Run this code. You should see "Luigi Mario" printed in your console. The first two variables contain the strings "Luigi" and "Mario", respectively. The + symbol, also called the addition operator, can be used to also combine strings. So when print(first_name + " " + last_name) runs, the two strings are put together to form a full name. Note the inclusion of the " ". That's not a mistake! What you see is actually an empty space, which is a string! It's included so that the two strings are more readable. You can remove the " " and one of the addition operators to see the result if you like!

Booleans

The last type of variable we'll discover are called Booleans. The good news about Booleans is that there are only two of them:
  1. True
  2. False
That's it! The slightly bad news is that Booleans can get confusing at times. Let's take a look at a few examples. is_raining = True is_sunny = False print(is_raining) # prints True print(is_sunny) # prints False This example is quite straight forward. The variable is_raining prints the value True because it's assigned a value of True. Because it's raining, then it's not sunny, so is_sunny = False. Now let's look at an example that might not be as obvious. on = 1 off = 0 print(bool(on)) # prints True print(bool(off)) # prints False In this example, you'll notice two things. First, we're assigning variables on and off values of 1 and 0, respectively. Then, you'll notice the bool() function. This function checks if the value is True or False and returns it to you. So, why then does 1 return True and 0 return False? In order to explain this, I'm afraid I'm going to need to use, math. Don't worry, it's quite intuitive. Just like True returns a value of True and False returns as a value of False, 0 and 1 return the same thing when checked with the bool() function.

Why? (Please don't say more math)

Actually, mathematics is one of the foundations of computer programming, but in this case I'll explain why using a simple electrical engineering analogy. Whenever you flip on a light switch, you're opening up an electrical current that travels down some wires and to the light bulb. But electrical hardware can't understand words like "on", so when a current is traveling, they instead use binary values, i.e, 1's and 0's. So when the light is on, it's because the elctrical current is signaling a value of 1, or True (to use humans). When the light switch is turned off, the current stops flowing, so the logic instead returns a value of 0, or False. As you progress in programming, you'll see this type of logic more frequently, specifically in the field of Boolean Algebra. We'll see some more examples in the next section.
If Else Statements
Since we discussed the topic of boolean logic briefly in the last section, it makes sense to discuss the next portion of logic called If/Else statements. If/Else statements are conditional logic statements that can help control the flow of our applications. The basic structure of if..else logic goes like this: If condition is true: Run code for when condition is true Else: Run code for when condition is false Lets take a look at a simple example, and then we'll examine a familiar but more complex example. is_raining = True if is_raining: print("It's raining. We can't go outside.") else: print("It's sunny!") Copy this code and run it in Thonny. When the code executes, the variable is_raining is set to True. Next, the if..else block is executed. Since it's raining, the print() function indented in the if block executes, returning the string indicating that it's raining. Now, try changing the value of is_raining to False. This time, the else block executes, because it's no longer raining. Pay special attention to the structure and syntax of the if..else block. In Python, if..else blocks contain the keyword if, followed by the condition, and ends with the : (colon) to indicate the beginning of the block. You also should pay attention to the indentation of the code running beneath the if and else blocks, respectively. In fact, try running the code after backspacing either print statement one time. You'll get an indentation error. Python forces you to indent code beneath if..else blocks, loops, and function declarations. Python is big on readable code and you will encounter this error quite often. It's annoying at first, but when you get used to it, you'll encounter it less. You might even appreciate it if you learn other programming languages which may have different formatting requirements.
Elif and Operators
Let's take a look at the code below. Then, we'll dive a little deeper. It's not a particularly useful program, but I wrote this to demonstrate a point about flow control (if..else) logic that deserves special attention (and also because I was really confused by this when I started learning.) # A simple program to determine if an integer is positive or negative def number(x): if x > 0: print("Positive") if x < 0: print("Negative") else: print("Zero") number(1) # Returns Positive and Zero You might be able to figure out what this code is doing simply by reading the lines starting with a # symbol. The # symbol means that the text contained on that line number is a comment. They're like notes a programmer leaves for themselves or for other programmers that might describe how something works. They are ignored whenever the program is running. This code determines if a number is positive or negative. Try it out! You can insert any number that you want within the parentheses of the number() function. Try a negative number, or 0, or 1,599,642. It will print a message in the console. Notice anything weird? Whenever you type a positive number, it returns two responses, "Positive" and "Zero". Weird, right? You also might notice the < and > symbols Those are called comparison operators. In Python, operators are special symbols or keywords that can help perform calculations or make logical decisions such as comparing two results. There are six comparison operators:
Operator Name Example
> Greater than x > y
< Less than x < y
>= Greater than or equal to x >= y
<= Less than or equal to x <= y
== Equal x == y
!= Not equal x != y
You may have encountered some of these before in a math class. The first four operators are called inequalities in mathematics. The integer 2 is less than 4, but that's mathematically expressed: 2 < 4. Logical operators in Python work the same way. Here's a quick script to illustrate. x = 5 # You can change x and y to whatever integers you want. y = 10 if x < y: print("Less than.") elif x > y: print("Greater than.") elif x >= y: print("Greater than or equal to.") elif x <= y: print("Less than or equal to.") else: print("More to come") The last two operators, == and !=, allow us to compare if two objects (or variables) contain the same values or not, respectively. Here's a quick example. x = 5 y = 5 if x == y: print("x is equal to y.") else: print("x is not equal to y") Since x and y both contain the integer 5, the print statement beneath the if block will execute. The else block will execute should x and y contain different values.

What's the difference between = and == ?

The = and == operators, though they might appear similar at first thought, do not mean the same thing! The difference lies in how they function. I'll illustrate using some code: # EXAMPLE 1 player_1 = "Lionel Messy" # "Lionel Messi" is assigned to first_name player_2 = "Cristiano Ronaldo" # "Cristiano Ronaldo" assigned to second_name # EXAMPLE 2 # Compare if player_1 and player_2 contain the same values! if player_1 == player_2: print("The same!") else: print("Not the same!")
In exmaple 1, the = operator is called the assignment operator. It assigns whatever value is to the right of the = operator and creates space for it in your computer's memory. The == is called the comparison operator, and it compares whether or not two objects contain the same values. In example 2, the if block compares if player_1 has the same value as player_2. Remember that player_1 contains the string "Lionel Messi" and player_2 is assigned the string "Cristiano Ronaldo". They are not the same strings, the code beneath the else block will execute.
